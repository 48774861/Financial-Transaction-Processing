# Real-Time Financial Transaction Data Processing for Fraud Detection Systems
Andrew Keuhn, Spencer Neill, Marc Pham  
## Abstract


## Introduction
   Modern financial platforms like banks and fintech applications must process millions of transactions per minute before fraud detection models can even begin their work. The core challenge is not just detecting fraud, but doing so quickly enough for banks to take preventative action in real time. To achieve this, systems must preprocess and organize transaction data at high speed so that machine learning models and downstream analytics can run without delay. This preprocessing stage includes sorting transactions by user and timestamp, merging multiple incoming data streams, and structuring the data for efficient analysis. Optimizing this step is critical, because faster data readiness leads directly to quicker fraud detection, reduced financial risk, and better scalability as transaction volumes continue to grow.  
   
   In this project, we developed a high-performance data processing pipeline using CUDA and GPUs. The transaction dataset was first divided into smaller, independent batches of about one million transactions each, reflecting real-world ingestion rates. To improve memory efficiency on the GPU, we stored the data in a Struct-of-Arrays (SoA) format, where each attribute (such as timestamp or user ID) is kept in its own separate array. Each batch was then sorted by user ID and timestamp using a parallel radix sort, which processes data digit by digit and is well-suited for parallel execution. After sorting the batches independently, we applied a parallel merge to combine them into a single, globally sorted dataset. Our objective was to outperform CPU-based systems like Apache Spark (an industry-standard tool for large-scale data processing) by leveraging GPU parallelism to further accelerate transaction preprocessing.  
   
   The use of parallel computing is justified by the inherently data-parallel nature of the problem. Radix sorts and merging can be decomposed into independent operations across large datasets, making them ideal for GPU execution. While CPU-based systems like Apache Spark are already highly optimized and perform very well at large-scale data processing, their performance is still limited by the number of CPU cores and general-purpose hardware constraints. Moving these operations to GPUs using CUDA can significantly increase our throughput by taking advantage of thousands of lightweight threads running in parallel. In financial systems where decisions must be made in milliseconds, even minor speed improvements can significantly enhance the performance and responsiveness of downstream processes like fraud detection.
